# Episode 03 Production Package — Duplication Reveals Abstraction

## Purpose
This package supports the episode:
**What "Duplicate" Code is Actually Trying to Tell You**

The demo illustrates:
- redundancy in intent across multiple scheduling methods
- redundancy appearing in tests
- collapsing repeated intent into a shared abstraction
- Template Method emerging naturally

## Files
- `meeting_coordinator_before.py` — redundant scheduling methods
- `scheduling_process_after.py` — shared abstraction after refactor
- `test_episode_03.py` — runnable pytest tests for before/after
- `UML_before_after.md` — conceptual before/after diagrams
- `shot_list.md` — screen capture plan and effects
- `production_package_template.md` — reusable template

## Run
```bash
pytest -q
```

## Capture notes
- Use VS Code dark theme
- Zoom so code is approximately 30–34px equivalent
- No scrolling during a shot
- Prefer static screenshots or frozen code clips
- Hold 2 seconds before/after each shot
